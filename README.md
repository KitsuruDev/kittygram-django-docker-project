## СТРУКТУРА ПРОЕКТА:

1. kittygram - основа проекта (настройки, пути, manage.py);

2. api - весь функционал со всеми данными.


## ОСНОВНЫЕ ШАГИ ИНИЦИАЛИЗАЦИИ (ЛОКАЛЬНОЕ РАЗВЁРТЫВАНИЕ):

1. Получить весь проект;

2. Создать виртуальное окружение в корне проекта с именем venv (python -m venv venv);

3. Установить в окружение всё из requirements.txt (pip install -r requirements.txt).


## ОСНОВНЫЕ ШАГИ ИНИЦИАЛИЗАЦИИ (ЛОКАЛЬНОЕ РАЗВЁРТЫВАНИЕ НА ОБРАЗЕ DOCKER):

1. Получить весь проект;

2. Собрать образы docker в корне проекта командами docker-compose build

3. Запустить все собранные образы docker-compose up

Либо сделать сборку и поднятие образов одной командой docker-compose up --build

Обязательно опустить все собранные образы (docker-compose down) и удалить их по завершении работы с проектом

Проект доступен по url-адресу http://localhost


## Очистить БД и загрузить тестовые данные со сгенерированными изображениями
python manage.py load_test_data

## Очистить БД (флаг force - без подтверждения)
python manage.py clear_database --force

## Запустить тест проверки функционала приложения (api) (тест сам создаёт для себя тестовые данные и очищает их)
python manage.py test api

## Запуск бекенд-части проекта:
python manage.py runserver

## Запуск фронтенд-части проекта (локально не запустится из-за путей зависимостей):
python -m http.server 3000
